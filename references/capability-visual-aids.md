# Capability D — visual aids

On-brand visual guidance within the hexagon system. Anything needing a real design tool is delivered
as **on-brand direction, not a finished raster.** Fill the `assets/visual-aids/*.md` skeletons.

## The brand visual system (applies to all three)

- Built on the **hexagon grid** (0/60/120°); each illustration/icon carries **one distinct idea**,
  never decoration. If it's only decorative, use nothing.
- **One color per icon.** Approved combos only: **Soft Black on white, Highlight Yellow on Soft
  Black, Soft Black on Highlight Yellow.**
- **Work Sans**, sentence case, left-aligned. A **single yellow highlight** per view, max. Never
  yellow text on white or white on yellow; never a full yellow canvas.
- **Products and exposure-management concepts are shown via hexagon illustration, never
  photography.** Photography (if any) is high-contrast black & white on a neutral backdrop — never
  for products/concepts.

## Diagram (`visual-aids/diagram-spec.md`)

A hexagon-grid diagram spec: the one idea, the nodes, the flow/relationship, color treatment, labels,
alt text. A brief for the art, not the art.

## Screenshots (`visual-aids/screenshot-guide.md`)

Which screens to capture (the ones that prove value); **blur/hide customer data**; **no competitor
UI** (would trip the brand+legal gate); annotate only in a single yellow accent; 1080p+; alt text.

## Social / OG card (`visual-aids/card.md`)

Headline + optional sub-line + hexagon-illustration visual; Soft Black base, White text, at most one
yellow accent; 1200×630 default; alt text. On-brand layout direction the contributor or a designer
executes.
